# Support

Use this guide to choose the right place for help.

## Where To Go

### Bug reports

Open a GitHub issue using the **Bug report** template when:

- something is broken
- a documented workflow no longer works
- behavior regressed
- the scheduler, web UI, adapter, or HCL engine behaves unexpectedly

### Feature requests

Open a GitHub issue using the **Feature request** template when:

- you want a new capability
- you want a new HCL feature
- you want a web UI enhancement
- you want a developer-experience improvement

### Mapping or source requests

Open a GitHub issue using the **Mapping or adapter request** template when:

- you want support for a new source system
- you need a new example mapping
- you think a dedicated source adapter is needed

### Security concerns

Do **not** file public issues for security problems. Use [SECURITY.md](SECURITY.md).

## Before Opening An Issue

Please try to include:

- the command you ran
- whether you used CLI, scheduler, Docker, or the web UI
- the relevant mapping file or HCL block
- logs or traceback
- your Python version and environment details
- the NetBox version, if relevant

Redact secrets before posting.
